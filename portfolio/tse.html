TINY SEARCH ENGINE


This is a search engine that crawls coded in C which goes through archieved Dartmouth Computer Science pages to find pages that are most relevant based on keywords. The search engine is split into three key components: crawler, indexer, and query.

Crawler is a standalone program which takes a seed webpage and crawls through all of subsequent websites. It starts with the seed page, parsing the HTML for embedded URLs, and continues until URLs from all pages have been accounted for (up to a specified depth). Each crawled page is saved into a document with the URL, index, and HTML content.

Indexer uses the previously crawled pages to create an index.dat file, which contains the occurrences of words in each file. It ultimately creates a list of webpages where a certain word occurs and indicates the frequency of the world (i.e. computer 1 5 2 17 - computer occurs in pages 1 and 2, with 5 and 17 instances respectively).

Query takes the index.dat file generated from indexer and the crawled files generated from crawler to create a command-line search. Query parses the user query appropriately (taking into consideration false words, capitalization, AND and OR commands, etc.), and then searches for the documents that include the highest frequencies of the search input. The documents and URLs are then displayed in a sorted list.
